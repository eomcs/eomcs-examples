# app06. 리액트 컴포넌트 인스턴스화

## 개요

지금까지 컴포넌트를 **선언**하는 방법(예: `function List() { ... }`)과, 그것을 **인스턴스화**하는 방법(예: `<List />`)을 배웠다. 이번 실습에서는 자바스크립트의 클래스 문법에 비유하여 리액트 컴포넌트를 선언하고 인스턴스화 하는 것을 개념을 이해한다.

## `App` 컴포넌트

이전 실습에서 만든 아래 코드에서 시작한다.

```jsx
// src/App.jsx
function App() {
  return (
    <div>
      <h1>My Hacker Stories</h1>

      <Search />

      <hr />

      <List />
    </div>
  );
}

function Search() { /* ... */ }

function List() { /* ... */ }

export default App;
```

## 실습

### 실습 1. 자바스크립트 클래스로 "선언 vs 인스턴스화" 복습하기

클래스는 주로 객체지향 프로그래밍 언어에서 사용된다. 자바스크립트는 함수형과 객체지향 패러다임이 함께 공존하는 멀티 패러다임 언어이기 때문에 클래스도 지원한다. 아래 `Person` 클래스를 예로 살펴보자.

```js
class Person {
  constructor(firstName, lastName) {
    this.firstName = firstName;
    this.lastName = lastName;
  }

  getName() {
    return this.firstName + ' ' + this.lastName;
  }
}
```

- 클래스는 인스턴스를 생성할 때 인자를 받아 대입해주는 **생성자(constructor)**를 가진다.
- 클래스는 인스턴스에 연결된 함수(`getName()` 같은)도 정의할 수 있는데, 이를 **메서드(method)**라고 부른다.
- `class Person { ... }` 처럼 클래스를 **선언(declaration)**하는 것과, `new` 키워드로 그 클래스의 **인스턴스(instance)**를 만드는 것은 서로 다른 별개의 작업이다. 클래스 선언은 설계도이고, 실제 사용하려면 `new`로 설계도에 따라 인스턴스를 만들어야 한다.

```js
// 클래스 선언
class Person { /* ... */ }

// 클래스 인스턴스화
const robin = new Person('Robin', 'Wieruch');
console.log(robin.getName());
// "Robin Wieruch"

// 또 다른 인스턴스화
const dennis = new Person('Dennis', 'Wieruch');
console.log(dennis.getName());
// "Dennis Wieruch"
```

하나의 클래스 **선언**으로부터 여러 개의 **인스턴스**를 만들 수 있다는 점이 핵심이다.

### 실습 2. 리액트 컴포넌트와 비교해보기

리액트 컴포넌트도 마찬가지다. 컴포넌트 **선언**은 하나뿐이지만, 그 컴포넌트의 **인스턴스**는 여러 개 만들 수 있다.

```jsx
// src/App.jsx

// App 컴포넌트 선언
function App() {
  return (
    <div>
      {/* ... */}

      {/* List 컴포넌트의 인스턴스 생성 */}
      <List />
      {/* List 컴포넌트의 또 다른 인스턴스 생성 */}
      <List />
    </div>
  );
}

// List 컴포넌트 선언
function List() { /* ... */ }
```

- 함수를 통해 컴포넌트를 한 번 정의하고 나면, JSX 어디에서든 엘리먼트(예: `<List />`)로 사용할 수 있다. 
- 이런 엘리먼트가 바로 컴포넌트의 **인스턴스**를 만들어내는 것이며, 이를 **"컴포넌트가 인스턴스화된다"** 고 표현한다. 
- 컴포넌트 **선언**이 존재하는 한, 원하는 만큼 얼마든지 인스턴스를 만들 수 있다.

| 개념 | 자바스크립트 클래스 | 리액트 컴포넌트 |
| --- | --- | --- |
| 선언(설계도) | `class Person { ... }` | `function List() { ... }` |
| 인스턴스화 | `new Person(...)` | `<List />` |
| 인스턴스 개수 | 원하는 만큼 여러 개 | 원하는 만큼 여러 개 |

## 현재 코드의 문제점

만약 `list` 변수를 계속 전역(컴포넌트 바깥)에 정의한 채로 둔다면, `List` 컴포넌트의 인스턴스를 여러 개 만들더라도 모든 인스턴스가 **같은 `list` 데이터**를 공유하게 된다. 각각의 `List` 컴포넌트 인스턴스가 서로 다른 데이터를 갖게 하려면 어떻게 해야 할까? 

다음 실습에서 배울 **Props**를 사용하면, **클래스 인스턴스화 시 생성자에 인자를 넘기는 것처럼 컴포넌트 인스턴스마다 서로 다른 데이터를 전달할 수 있다.**

## 정리

- 컴포넌트 **선언**은 한 번만 하지만, JSX에서 엘리먼트로 사용할 때마다 새로운 **인스턴스**가 만들어진다.
- 이는 자바스크립트 클래스를 `class`로 선언하고 `new`로 여러 인스턴스를 만드는 것과 비슷한 감각이다. (단, 기술적으로 둘은 무관하다.)
- 지금은 `list` 데이터가 컴포넌트 바깥의 전역 변수라서 모든 `List` 인스턴스가 데이터를 공유한다는 한계가 있으며, 이는 다음 실습(Props)에서 해결한다.
