# Chapter 12: 창발성 (Emergence)

이번 장에서는 창발성에 대해 다룬다. 좋은 설계는 한 번에 만들어지는 것이 아니라, 작은 개선과 리팩토링을 반복하면서 자연스럽게 드러난다.

## 예제

- 창발적 설계로 깔끔한 코드를 구현하자 (Getting Clean via Emergent Design)
- 단순 설계 규칙 1: 모든 테스트를 실행하라 (Simple Design Rule 1: Runs All the Tests)
- 단순 설계 규칙 2–4: 리팩토링 (Simple Design Rules 2–4: Refactoring)
- 중복을 없애라 (No Duplication)
- 표현하라 (Expressive)
- 클래스와 메서드 수를 최소로 줄여라 (Minimal Classes and Methods)
