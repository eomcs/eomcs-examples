# Chapter 9. 단위 테스트 (Unit Tests)

이번 장에서는 단위 테스트를 작성하고 실행하는 방법을 소개한다. 단위 테스트란 코드의 작은 단위(주로 함수나 메서드)가 의도한 대로 동작하는지 검증하는 테스트를 말한다.

  - 테스트는 유연성, 유지보수성, 재사용성을 제공한다 (Tests Enable the -ilities)
- [깔끔한 테스트 (Clean Tests)](exam03/README.md)
  - 도메인에 특화된 테스트 언어 (Domain-Specific Testing Language)
  - 이중 표준 (A Dual Standard)
- [테스트에 assert 하나 (One Assert per Test)](exam04/README.md)
  - 테스트 당 개념 하나 (Single Concept per Test)
- [F.I.R.S.T (Fast, Independent, Repeatable, Self-validating, Timely)](exam05/README.md)
