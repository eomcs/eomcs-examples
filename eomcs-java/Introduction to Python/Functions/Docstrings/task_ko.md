## 독스트링 (Docstrings)

한 줄로 작성된 독스트링은 매우 명확한 경우에 사용됩니다. 여러 줄로 구성된 독스트링은 한 줄 독스트링처럼 요약 문장으로 시작하고, 빈 줄 뒤에 추가적인 설명이 이어집니다.  
함수나 메서드에 대한 독스트링은 해당 함수나 메서드의 동작을 요약하고, 매개변수, 반환 값(들), 부작용, 발생하는 예외, 호출 가능한 조건 등의 내용을 문서화해야 합니다 (해당되는 모든 항목 포함). 선택적인 매개변수는 명시해야 합니다. 키워드 매개변수가 인터페이스의 일부인지 여부도 문서화해야 합니다.  

독스트링은 일반적으로 모듈, 클래스 및 메서드 정의에도 작성해야 합니다 (이러한 내용은 이후 강좌에서 배우게 될 것입니다). 독스트링 작성 규칙에 대한 자세한 내용은 [Python PEP Guide](https://peps.python.org/pep-0257/)를 참조하세요.

더 구조화되고 상세한 정보는 [Hyperskill 지식 베이스 페이지](https://hyperskill.org/learn/step/11869?utm_source=jba&utm_medium=jba_courses_links)에서도 확인할 수 있습니다.
